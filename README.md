# SecureGate AI - Container Image Vulnerability Scanner with Reporting

**An Enterprise DevSecOps Security Orchestration Platform**

<br>

## Problem Statement

Modern applications are increasingly deployed using containers. While containers provide portability and scalability, they can also introduce security risks if vulnerable software packages are included in the container image.

## Solution Overview

**(How SecureGate AI solves the problem)**

SecureGate AI addresses these challenges by integrating security directly into the CI/CD pipeline. It automatically performs security scans, vulnerability assessments, risk analysis, and policy enforcement before container images are published or deployed, ensuring that only secure and compliant artifacts progress through the software delivery lifecycle.

The objective of this project is to build an automated Container Image Vulnerability Scanner that identifies known vulnerabilities before container images are deployed to production environments.

### Objectives
   - Detect vulnerabilities in container images
   - Automate security checks in CI/CD pipelines
   - Generate risk scores and reports
   - Enforce deployment approval policies
   - Provide centralized visibility through dashboards

----

## Elevator Pitch:
  
**SecureGate AI** is a centralized DevSecOps platform that integrates with GitHub Actions and Jenkins through APIs, automatically performs source code security analysis, secret detection, filesystem scanning, code quality assessment, container image vulnerability scanning, risk scoring, report generation, approval/rejection decisions, notifications, and dashboard analytics before allowing container images to be pushed to Docker Hub.

SecureGate AI is not only a Container Image Vulnerability Scanner, but it's a complete Enterprise-level DevSecOps Security Orchestration Platform.

## CI/CD Flow

### Successful Pipeline Workflow

1. When a developer pushes code, the CI/CD pipeline automatically triggers **SecureGate AI** through an API call.
2. The platform performs secret detection, filesystem scanning, code quality analysis, Docker image vulnerability scanning, risk scoring, and report generation.
3. If all security checks pass and the security score meets the approval criteria, the Docker image is pushed to Docker Hub, and
4. A deployment approval report is generated, and notifications are sent via Slack and email.

### Failed Pipeline Workflow

1. When a developer pushes code, the CI/CD pipeline triggers **SecureGate AI** to perform comprehensive security scans and quality checks.
2. If critical vulnerabilities, exposed secrets, quality gate failures, or insufficient security scores are detected, the deployment process is immediately stopped.
3. The platform generates a detailed failure report with findings, remediation recommendations, and security improvement actions, and then
4. Sends real-time alerts through Slack and email to notify the development and security teams.

## 2. FastAPI (Backend API)

### Purpose

FastAPI acts as the central orchestration engine of the platform.

### Responsibilities

- Report APIs

## 3. SonarQube

### Purpose

SonarQube performs Static Application Security Testing (SAST) and code quality analysis.

It identifies security vulnerabilities, code smells, bugs, duplicated code, maintainability issues, and security hotspots.4

### Responsibilities

- Code Smells Detection
- Security Hotspots
- Bug Detection
- Technical Debt Analysis
- Quality Gate Validation
- Code Coverage Analysis

## 4. GitLeaks

GitLeaks scans source code repositories for exposed secrets and credentials.

## 8. Risk Scoring Engine (Custom Module)

### Purpose

A custom module developed within the project that calculates a security score based on all scan results.

### Why?

Individual tools generate separate reports.

The Risk Engine consolidates all findings into a single security score.

### Responsibilities

- Security Score Calculation
- Risk Prioritization
- Pass/Fail Evaluation
- Deployment Readiness Assessment
